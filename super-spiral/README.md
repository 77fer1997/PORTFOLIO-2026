# Luis Calcina — Creative Portfolio

Static, editorial portfolio connecting frontend development, visual design and acting under the idea “Code, image and presence.” Built with semantic Astro components, typed content and progressively enhanced motion.

## Stack

Astro 7 SSG, strict TypeScript, Tailwind CSS 4, Astro Content Collections, GSAP + ScrollTrigger, Lenis, Three.js, View Transitions and Astro Assets.

## Setup

Requires Node.js 22.12 or newer.

```bash
npm install
npm run dev
npm run build
npm run preview
npm run astro check
```

The production output is generated in `dist/`. No server adapter, API route or runtime database is used.

## Structure

- `src/components`: global shell, sections, project views and Three.js host.
- `src/content/projects`: Markdown case studies with validated frontmatter.
- `src/content/experience`: typed timeline entries.
- `src/data`: editable identity, navigation, disciplines and skills.
- `src/layouts`: document metadata, accessibility shell and global UI.
- `src/pages`: static pages and project routes generated by `getStaticPaths()`.
- `src/scripts`: centralised browser behaviour and motion lifecycle.
- `src/assets/images`: source artwork processed by Astro Assets.

## Editing identity and content

Update personal information and links in `src/data/site.ts`. The placeholder name, email, profile links and production URL should be replaced before launch. Update the `site` property in `astro.config.mjs` and matching URL in `public/robots.txt`.

To add a project, duplicate a Markdown file in `src/content/projects`, change every required frontmatter field and point `cover` to a local image under `src/assets/images`. The schema in `src/content.config.ts` validates builds and the dynamic route is generated automatically. Use descriptive alternative text and source images large enough for their display size; Astro produces AVIF/WebP variants and responsive widths.

## Contact form

Create `.env` from your deployment settings and set a public, non-secret endpoint:

```env
PUBLIC_CONTACT_FORM_ENDPOINT=https://formspree.io/f/your-id
```

Without an endpoint, the form falls back to the visitor's email client. With an endpoint it retains standard HTML submission and adds fetch-based status feedback, native validation and a honeypot.

## Accessibility and performance

The site works without JavaScript and without WebGL. It includes a skip link, semantic landmarks, keyboard-accessible navigation, focus styles, labelled fields, live form messages and global reduced-motion behaviour. Three.js is dynamically imported near the viewport, capped for pixel density and disabled on mobile/reduced motion. Lenis is omitted for touch and reduced-motion users.

## Deployment

Use build command `npm run build`, publish directory `dist`, and Node 22.12+ on Vercel, Netlify or Cloudflare Pages. GitHub Pages may require setting Astro's `base` when publishing below a repository subpath. Configure the custom domain in the host, update the canonical site URL, and rebuild whenever collection content changes.
